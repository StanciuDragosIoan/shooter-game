update(dt, now) – What each part does

function update(dt, now) {
  // move player
  // shoot (with cooldown)
  // move bullets
  // move enemies + enemy shooting
  // bullets vs enemies (collision)
  // wave progression
}
  • Player movement: checks keys and updates player.x. Then clamps so you can't leave the screen.
  • Shooting: only fires if enough time passed since last shot (now - shootTimer > cooldown).
  • Bullets update: each bullet moves by dy, and filter removes ones off-screen.
  • Enemies: each enemy moves down; randomly fires using Math.random() < shootRate.
  • Collision: for each bullet, check against enemies using hits. On hit → remove enemy (splice) and remove bullet (return false).
  • Wave progression: when no enemies remain, wait a bit, then increase wave and call spawnWave.

Key idea: update is the "brain" that runs every frame. It changes the game state; draw just shows it.

Try: Hold Space to see cooldown, clear all enemies to trigger next wave, and watch bullets being removed by filter.
